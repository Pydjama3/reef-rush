Le But

(NOTE: la version française des règles n'est pas à jour...)

Le but de ce jeu est de gagner.

Pour cela, vous devrez contrôler un sous-marin pour collecter les déchets plastiques sur les coraux. Attention de ne pas vous noyer !

  Règles

Le jeu se joue tour par tour.

Votre programme reçoit 3 informations différentes :
  • Votre niveau d'oxygène actuel
  • La quantité de déchets plastiques à votre poste
  • La sortie du sonar à bord de votre sous-marin

En réponse à ces informations, vous devrez émettre une commande: none (equivalent of no command), up, right, down, or left.

The game zone works as follows:
  • L'élément principal du jeu est la carte générée de manière procédurale dans laquelle les joueurs peuvent évoluer
  • La carte est une carte de tuiles 2D composée de blocs/entités solides (murs de sable, sous-marins, surface de l'eau) et de blocs creux (eau, coraux).
  • La taille de la carte n'est pas donnée au début du jeu
  • La carte est toujours symmétrique

Le sous-marin fonctionne comme suit :
  • Ils se déplacent et ne peuvent pas traverser d'autres sous-marins
  • Ils collectent automatiquement les déchets plastiques s'il y en a à leur poste
  • Votre niveau d'oxygene diminue si vous êtes sous la surface
  • Ils reconstituent automatiquement leur capacité en oxygène lorsqu'ils sont à la surface
Vous perdez si :
  • Vous vous trompez.
  • Vous ne fournissez pas une séquence d'actions valide.
  • Vous vous noyez (oxygene < 0)

  Note

N'oubliez pas de lancer les tests en les lançant depuis la fenêtre « Cas de tests ». Vous pouvez soumettre à tout moment temps pour recevoir un score contre les validateurs de formation. Vous pouvez soumettre autant de fois que vous le souhaitez. Ton la plupart la soumission récente sera utilisée pour le classement final.

  Entrée du jeu

Le programme doit d'abord lire les données d'initialisation à partir de l'entrée standard. Ensuite, fournissez à le sortie standard une ligne avec une commande (détaillée ci-dessous).
Input

Ligne 1 : un int oxygène, le courant

Ligne 2 : un int plasticCount, la quantité de plastique à la position du sous-marin

Puis 4 lignes pour le sonar

Ligne 3 : une chaîne de charactères y+, formatée de cette façon : y+=[WALL | SUBMARINE | CORAL | SURFACE]([0-2⁷]m)

Ligne 4 : une chaîne de charactères x+, formatée de cette façon : x+=[WALL | SUBMARINE | CORAL | SURFACE]([0-2⁷]m)

Ligne 5 : une chaîne de charactères y-, formatée de cette façon : y-=[WALL | SUBMARINE | CORAL | SURFACE]([0-2⁷]m)

Ligne 6 : une chaîne de charactères x-, formatée de cette façon : x-=[WALL | SUBMARINE | CORAL | SURFACE]([0-2⁷]m)

Output

Une seule ligne contenant l'une des commandes suivantes (non sensible à la casse) : NONE, UP, RIGHT, DOWN ou LEFT.


Constraints
0 ≤ oxygene ≤ 42
0 ≤ plasticCount ≤ 20
13 ≤ {y+|x+|y-|x-} ≤ 18
0 ≤ distance ≤ 2⁷

Temps de réponse alloué à la sortie est ≤ 2 secondes.
