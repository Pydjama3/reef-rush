The Goal

The aim of this game is to win.

For this you'll have to control a submarine to collect plastic waste on corals. Be careful to not drown !

  Rules

The game is played turn by turn.

Your program receives 3 different inputs:
  • Your current oxygen level
  • The amount of plastic waste at your position
  • The output of the sonar onboard your submarine

In response to these information, you need to output a command: NONE (equivalent of no command), UP, RIGHT, DOWN, or LEFT.

The game zone works as follows:
  • The game zone is a procedurally generated map in which the players can move
  • The map is a 2d tilemap made out of solid blocks/entities (sand walls, submarines, water surface) and hollow blocks (water, corals).
  • The size of the map is not given
  • The map is always symmetric
  • The x-coordinates of the increase towards the right, the y-coordinates increase towards the top.

The submarine work as follows:
  • They can't go on solid tiles (sand walls, submarines, water surface) but they can go on corals.
  • They move around and cannot go through other submarines
  • They automatically collect plastic waste if there is any at their position
  • Your oxygen level decreases by 1 at each turn if your submarine is under the surface
  • They automatically fully replenish their oxygen capacity when at the surface (on the highest row of tiles).
Victory Conditions:
  • You have more points than your opponent
  • You are the only active player left
Defeat Conditions
  • Your bot times out.
  • You drown (oxygen ≤ 0)

  Expert Rules

  • The properties WIDTH and HEIGHT allow you define the map's size.
  • WIDTH should be an even number
  • The area of the map shouldn't exceed the area of a 2⁶x2⁵ map
  • The parameter MAX_OXYGEN allows you to define the max oxygen capacity of the submarines during the game
  • The parameter MAX_TURNS allows you to define the number of rounds during which the game will be played

  Game Input

Input

Line 1: An Integer oxygen, the current oxygen level

Line 2: An Integer plasticCount, the amount of plastic at the submarine's position

Then 4 lines for the sonar output in this format:

[y+ | x+ | y- | x-]=[WALL | SUBMARINE | CORAL | SURFACE](distancem)

Line 3: A string y+ for data on the block or entity directly [below/above (choose one)] your current position.

Line 4: A string x+ for data on the block or entity directly to the right of your current position.

Line 5: A string y- for data on the block or entity directly [below/above (choose one)] your current position.

Line 6: A string x+ for data on the block or entity directly to the left of your current position.

Output

A single line containing one of the following commands (not case sensitive): NONE, UP, RIGHT, DOWN, or LEFT.

Constraints
0 ≤ oxygen ≤ 42
0 ≤ plasticCount ≤ 20
0 ≤ distance ≤ 2⁷

Allotted response time to output is ≤ 2 seconds.
