The Goal

The aim of this game is to win.

For this you'll have to control a submarine to collect plastic waste on corals. Be careful not to drown !

  Rules

The game is played turn by turn.

Your program receives 3 different inputs:

  • Your current oxygen level
  • The amount of plastic waste at your position
  • The output of the sonar onboard your submarine

In response to these inputs, you need to output a command: NONE (equivalent of no command), UP, RIGHT, DOWN, or LEFT.

The game zone works as follows:

  • The game zone is a procedurally generated map in which the players can move
  • The map is a 2d tilemap made out of solid blocks/entities (sand walls, submarines, water surface) and hollow blocks (water, corals).
  • The size of the map is not given
  • The map is always symmetric
  • The x-coordinates increase towards the right, the y-coordinates increase towards the top.

The submarines work as follows:

  • They can't go on solid tiles (sand walls, submarines, water surface) but they can go on corals.
  • They move around and cannot go through other submarines
  • They automatically collect plastic waste (1 unit each turn) if there is any at their position
  • Your oxygen level decreases by 1 each turn if your submarine is under the surface
  • They automatically fully replenish their oxygen capacity when at the surface (on the highest row of tiles).
Victory Conditions
  • You have more points than your opponent.
  • You are the only active player left.
Defeat Conditions
  • Your bot times out.
  • You drown (oxygen = 0).

  Expert Rules

How the game works normally:

  • The size of the map is 2^n in WIDTH by 2^(n-1) in HEIGHT, with n a random integer betweeen 4 included and the sum of 4 and the league level excluded. For example, if the league level is 1 n = 4; if the league level is 3, n=4, n=5 or n=6
  • The MAX_OXYGEN capacity is defined by the league level: 21 if the league level is 1, 42 if it is 2, and 63 if it is 3
  • The MAX_TURNS (sum of the players' turns) is defined by the league level: 100 if the league level is 1, 300 if it is 2, and 500 if it is 3

What you can configure and how:

  • The properties WIDTH and HEIGHT allow you define the map's size.
  • WIDTH should be an even number
  • The area of the map shouldn't exceed the area of a 2⁶x2⁵ map (approximately the max buffer size)
  • The parameter MAX_OXYGEN allows you to define the max oxygen capacity of the submarines during the game.
  • The parameter MAX_TURNS allows you to define the number of rounds during which the game will be played

  Game Input

Input

Line 1: An Integer oxygen, the current oxygen level

Line 2: An Integer plasticCount, the amount of plastic at the submarine's position

Then 4 lines for the sonar output in this format:

[y+ | x+ | y- | x-]=[WALL | SUBMARINE | CORAL | SURFACE](distancem)

Line 3: A string y+ for data on the block or entity above your current position.

Line 4: A string x+ for data on the block or entity to the right of your current position.

Line 5: A string y- for data on the block or entity below your current position.

Line 6: A string x+ for data on the block or entity to the left of your current position.

distance indicates the distance of your submarine from the first WALL, SUBMARINE, CORAL or SURFACE in the indicated direction of y+, x+, y- or x-.

Output

A single line containing one of the following commands (not case sensitive): NONE, UP, RIGHT, DOWN, or LEFT.

Note: outputting an invalid command or a direction which your submarine cannot move to has the same effect as outputting NONE.

Constraints
0 ≤ oxygen ≤ 42
0 ≤ plasticCount ≤ 20
0 ≤ distance ≤ 2⁷

Allotted response time to output is ≤ 2 seconds.
